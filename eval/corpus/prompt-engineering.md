# 프롬프트 엔지니어링 기초

## 시스템 프롬프트

대화 전체에 적용되는 최상위 지시. 역할("당신은 간결하게 답하는 한국어 어시스턴트"), 어조, 금지 사항을 정의한다. 요령:

- 짧고 구체적으로 — 장황한 규칙 목록보다 핵심 원칙 몇 개가 소형 모델에 효과적이다.
- 매 요청의 첫 메시지로 항상 포함한다 (stateless이므로).
- 소형 모델(8B급)은 복잡한 시스템 프롬프트의 일부를 무시할 수 있다 — 중요 지시는 사용자 메시지 쪽에 반복하는 것도 방법이다.

## Few-shot 예시

원하는 출력 형식의 예시를 1~3개 보여주면 지시문만 쓰는 것보다 형식 준수율이 크게 오른다. 분류·추출·포맷 변환 작업에서 특히 효과적이다.

## RAG 컨텍스트 주입 패턴

검색 발췌를 넣을 때의 정석 구조:

```
다음은 질문과 관련된 문서 발췌다. 답변에 활용하되, 관련이 없으면 무시하라.

[출처경로 > 섹션제목]
발췌 내용...
```

- **"관련 없으면 무시하라"**: 검색이 빗나갔을 때 모델이 억지로 발췌에 답을 끼워 맞추는 것을 줄인다.
- **출처 라벨**: `[경로 > 헤딩]` 형식으로 넣으면 모델이 답변에서 그 라벨을 그대로 인용하는 경향이 있다 — 출처 표시가 공짜로 생긴다.
- 발췌와 지시를 시스템 메시지로 넣으면 사용자 발화와 분리되어 모델이 "자료"로 취급하기 쉽다.

## 요약 프롬프트 패턴

대화 압축용 요약은 "무엇을 보존할지"를 명시해야 한다: "사용자가 알려준 사실·선호·결정 사항을 우선 보존하라". 그냥 "요약하라"고 하면 분위기 요약이 나와 이후 대화에서 구체 정보(이름, 숫자, 선택)가 소실된다.

## 소형 모델(8B급) 다루기

- 지시는 하나씩, 단계는 명시적으로. 복합 지시("A 하고 B 하되 C는 빼고")는 일부가 누락되기 쉽다.
- 출력 형식을 강하게 고정하고 싶으면 "색 이름만 답해"처럼 부정형이 아닌 긍정형 제약을 쓴다.
- 환각(지어내기)이 의심되는 답에는 "문서 근거로", "모르면 모른다고 답해"를 덧붙이면 완화된다 — 제거되진 않는다.
